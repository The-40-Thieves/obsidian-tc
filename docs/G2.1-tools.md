## r1 → r2 changes (revision summary)

Structural changes applied after cold-read review against locked G1/G2.1-conventions decisions:

- **Domain 1:** `reload_vault` split into `reload_vault` (config-only) + `reset_vault_cache` (destructive admin op). 3 → 4 tools.
- **Domain 2:** Five write tools (`create_note`, `write_note`, `append_note`, `prepend_note`, `patch_note`) collapsed to three (`write_note` with `mode`, `append_note` with `position`, `patch_note` unchanged). 12 → 10 tools.
- **Domain 3:** Four frontmatter write tools (`set_property`, `remove_property`, `merge_frontmatter`, `replace_frontmatter`) collapsed to one `update_frontmatter` with `mode`. 8 → 5 tools.
- **Domain 5:** `prune_hub_links` generalized — caller passes a `policy` object with named primitives + JSONLogic escape hatch. Skill-specific classifier removed from substrate.
- **Domain 2 (`copy_note`):** Default `rewrite_internal_links` flipped from `false` to `true` (matches `move_note`'s update-by-default stance, eliminates silent link breakage on cross-folder copy).
- **Domain 7 / 8 (`query_base`, `query_canvas`):** Rate-limit class promoted to `bulk` when source/scope can fan out across vault.
- **Domain 27 (`generate_uri`):** ACL changed from `admin` to scope-free (it's a pure string-builder utility, no vault state touched).
- **Conventions:** Added `auto_mode_explain` per-vault config option as forward-ref to G2.3/G2.4 — lets operators force explain emission for autonomous-agent vaults without re-litigating the per-call default.
- **Cross-cutting list:** Added item #8 (HITL threshold policy location → G2.2 Architecture).

Net tool count: 109 → 105. Operation coverage: identical. Pillar 1 ("comprehensive ~100 tools covering every meaningful Obsidian operation") preserved. (The shipped registry is 103: the per-domain column sums to 101, plus index_vault + server_health.)

## Conventions

**Naming.** Flat snake_case verb-first. Domain prefix where the noun is ambiguous (`bases_query` not `query`, `tasks_filter` not `filter`). Read-side verbs: `read`, `list`, `search`, `get`, `find`, `count`. Write-side verbs: `write`, `create`, `update`, `append`, `prepend`, `patch`, `delete`, `move`, `rename`. Bulk variants prefix `bulk_` (`bulk_write`, `bulk_tag`).

**Every tool that touches vault content takes `vault: string`.** Resolves against the multi-vault registry. Defaults to `OBSIDIAN_TC_DEFAULT_VAULT` env when omitted. Cross-vault ops take `vaults: string[]`.

**Pagination.** Any tool that can return >100 results uses cursor pagination: input `limit?: number (max 1000)`, `cursor?: string`. Output `items: T[]`, `cursor?: string` (omitted when no next page), `overflow?: boolean` (true when caller's `limit` was clamped).

**Idempotency keys.** Any non-idempotent write accepts `idempotency_key?: string`. Server stores keyed result for 24h in `idempotency_keys` table. Replay returns cached result.

**Mode-discriminated tools.** Where a single operation has multiple write semantics (`write_note`, `update_frontmatter`), the tool takes a `mode` field with mode-specific optional fields. The output includes `mode_used` so the dispatcher trace shows which path fired. Mode-specific HITL/idempotency rules documented per-tool.

**Explain flag.** Tools that make non-obvious routing decisions (`auto` mode search, plugin-bridged ops) accept `explain?: boolean` and return `_explain?: object` describing what fired.

**Auto-mode explain at vault scope.** Per-vault config (`auto_mode_explain: "always" | "on_zero_hits" | "optional"`, default `optional`) lets operators force `_explain` emission for vaults serving autonomous agents, without re-litigating the per-call default. Forward-ref: G2.3 config schema, G2.4 observability defaults.

## Standard primitives (shared Zod schemas)

```typescript
// packages/shared/src/schemas/primitives.ts

export const VaultId = z.string().min(1).max(64).regex(/^[a-z0-9_-]+$/);

export const VaultPath = z.string()
  .min(1)
  .max(1024)
  .refine(p => !p.includes('..'), 'path traversal rejected')
  .refine(p => !p.startsWith('/'), 'absolute paths rejected');

export const Pagination = z.object({
  limit: z.number().int().positive().max(1000).optional(),
  cursor: z.string().optional(),
});

export const PaginatedResponse = <T extends z.ZodTypeAny>(item: T) => z.object({
  items: z.array(item),
  cursor: z.string().optional(),
  overflow: z.boolean().optional(),
  total_count: z.number().int().optional(),
});

export const WriteOptions = z.object({
  idempotency_key: z.string().min(1).max(128).optional(),
  create_dirs: z.boolean().default(true),
  // Note: legacy `overwrite` field removed from WriteOptions in r2.
  // Mode-discriminated tools (write_note, update_frontmatter) use `mode` instead.
  // Tools that still need a boolean overwrite flag (create_base, create_canvas,
  // create_excalidraw) declare it inline.
});

export const ElicitToken = z.string().regex(/^elicit_[a-z0-9]{16}$/);
```

## Error taxonomy

All tool errors return MCP `isError: true` with a structured payload:

```typescript
{
  code: string,
  message: string,
  details?: object,
  retry_after_ms?: number
}
```

| Code | Meaning | Retryable |
|---|---|---|
| `vault_not_found` | `vault` parameter does not match registry | no |
| `note_not_found` | Path resolves to no file | no |
| `path_invalid` | Traversal, absolute, or normalization failed | no |
| `path_ambiguous` | Wikilink or partial match resolves to multiple files | no (suggests candidates) |
| `acl_denied` | Path outside `read_paths` / `write_paths` / `delete_paths` | no |
| `read_only_mode` | Kill switch active | no |
| `note_exists` | Write to existing path without permission (e.g. `mode=create` + exists) | no |
| `concurrent_modification` | Underlying file changed between read and write | yes (with re-read) |
| `invalid_input` | Zod validation failed | no |
| `bases_syntax_error` | `.base` YAML or filter syntax invalid | no |
| `dql_error` | Dataview DQL parse or runtime error | no |
| `jsonlogic_error` | JSONLogic shape invalid | no |
| `plugin_missing` | Required Obsidian plugin not detected | no |
| `plugin_unreachable` | Plugin detected but REST endpoint failed | yes |
| `embedding_provider_error` | Embedding backend failed | yes |
| `operation_timeout` | Tool exceeded its time budget | yes |
| `bulk_partial` | Bulk op had per-item failures (see `details.results[]`) | partial |
| `rate_limit` | Throttle hit | yes (after `retry_after_ms`) |
| `elicit_required` | HITL confirmation needed (see `details.elicit_token`) | yes (after elicit) |
| `internal_error` | Unexpected server-side failure | maybe |

## Tool spec template

```
tool_name                            (one-line MCP description)
  acl:       <read|write|delete|execute|admin> on <scope expression>
  hitl:      <never|optional|required>
  idem:      <pure|natural|keyed|non-idem>
  ratelimit: <read|write|bulk>
  input:     <Zod schema or extends primitive>
  output:    <Zod schema or extends primitive>
  errors:    <error codes that can fire>
  obs:       <span name + custom tags>
```

```
read_notes                           Batch-read up to 50 notes

  hitl:      never
  idem:      pure
  ratelimit: read
  input:     { vault, paths: string[] (max 50), include_frontmatter?, include_content? }
  output:    { results: Array<{ path, ok: boolean, note?, error? }> }
  errors:    invalid_input
  obs:       tags batch_size

list_notes                           List notes under a directory with filters
  acl:       read on root
  hitl:      never
  idem:      pure
  ratelimit: read
  input:     { vault, root?: VaultPath (default ""), recursive?: boolean (default true), include_folders?: boolean, extensions?: string[] (default [".md"]), ...Pagination }
  output:    PaginatedResponse({ path, type: "file"|"folder", size, mtime })
  errors:    path_invalid, acl_denied

note_exists                          Check existence without reading
  acl:       read on path
  hitl:      never
  idem:      pure
  ratelimit: read
  input:     { vault, path: VaultPath }
  output:    { exists: boolean, type?: "file"|"folder" }
  errors:    path_invalid

write_note                           Write a note's full content (create | overwrite | upsert modes)
  acl:       write on path
  hitl:      required if mode="overwrite" on a non-empty existing file
  idem:      keyed
  ratelimit: write
  input:     { vault, path: VaultPath, content: string, frontmatter?: object,
               mode: "create"|"overwrite"|"upsert" (default "create"),
               expected_hash?: string (only honored when mode="overwrite", enables CAS),
               idempotency_key?: string,
               create_dirs?: boolean (default true),
               elicit_token?: ElicitToken }
  output:    { path, mode_used: "create"|"overwrite"|"upsert",
               content_hash, prev_hash?: string,
               created_at?: string, updated_at?: string }
  errors:    note_exists (mode=create + exists),
             note_not_found (mode=overwrite + missing),
             concurrent_modification (expected_hash mismatch),
             acl_denied, read_only_mode, elicit_required
  obs:       tags mode_used

  Mode semantics:
    create:   fails with note_exists if path exists. Returns created_at.
    overwrite: fails with note_not_found if path missing. Triggers HITL if file non-empty.
              Honors expected_hash for compare-and-swap; returns prev_hash on success.
              Returns updated_at.
    upsert:   creates if missing, overwrites if exists. No HITL by default
              (callers wanting safety pass expected_hash). Returns created_at or updated_at
              and a boolean `was_created` in output.

append_note                          Append or prepend content to a note
  acl:       write on path
  hitl:      never
  idem:      keyed (required; without key, repeated calls duplicate inserts)
  ratelimit: write
  input:     { vault, path: VaultPath, content: string,
               position: "end"|"start" (default "end"),
               after_frontmatter?: boolean (default true, position="start" only;
                                            inserts after frontmatter block when present),
               ensure_newline?: boolean (default true),
               idempotency_key: string }
  output:    { path, position_used, updated_at, content_hash, inserted_bytes }
  errors:    note_not_found, invalid_input (missing idempotency_key), acl_denied, read_only_mode

patch_note                           Replace a unique string within a note (fails if string not found or appears >1 times)
  acl:       write on path
  hitl:      optional (required if replacing across heading boundaries)
  idem:      natural
  ratelimit: write
  input:     { vault, path, old_string: string, new_string: string, expected_occurrences?: number (default 1) }
  output:    { path, updated_at, occurrences_replaced, content_hash }
  errors:    note_not_found, invalid_input ("old_string not found" | "old_string ambiguous"), acl_denied

delete_note                          Soft-delete (move to .trash/) by default; hard-delete only with explicit flag
  acl:       delete on path
  hitl:      required
  idem:      natural
  ratelimit: write
  input:     { vault, path, hard?: boolean (default false), elicit_token?: ElicitToken }
  output:    { path, deleted_at, recoverable: boolean, trash_path?: string }
  errors:    note_not_found, acl_denied, read_only_mode, elicit_required

move_note                            Move/rename + update incoming wikilinks
  acl:       write on both old and new path
  hitl:      optional (required if crossing top-level folder boundary)
  idem:      natural
  ratelimit: write
  input:     { vault, from: VaultPath, to: VaultPath, update_backlinks?: boolean (default true), overwrite_target?: boolean (default false) }

copy_note                            Copy with optional path-rewriting of wikilinks in body

  hitl:      optional (required if target outside same folder)
  idem:      keyed
  ratelimit: write

  output:    { from, to, created_at, content_hash, links_rewritten: number }
  errors:    note_not_found, note_exists, path_invalid, acl_denied
```

```
update_frontmatter                   Update frontmatter (set | remove | merge | replace modes)
  acl:       write on path
  hitl:      required if mode="replace"
  idem:      natural (mode=set|remove|merge); keyed (mode=replace, idempotency_key recommended)
  ratelimit: write
  input:     { vault, path,
               mode: "set"|"remove"|"merge"|"replace",
               key?: string (required for mode=set|remove),
               value?: string|number|boolean|null|Array<string|number>|object (required for mode=set),
               patch?: Record<string, unknown> (required for mode=merge; null values delete keys),
               frontmatter?: object (required for mode=replace),
               idempotency_key?: string (recommended for mode=replace),
               elicit_token?: ElicitToken (required for mode=replace) }
  output:    { path, mode_used, updated_at, content_hash,
               prev_value?: unknown (set|remove),
               applied?: object (merge: keys+values set),
               removed?: string[] (merge: keys deleted via null),
               prev_frontmatter?: object (replace) }
  errors:    note_not_found, invalid_input (missing mode-required field), acl_denied, read_only_mode, elicit_required
  obs:       tags mode_used

  Mode semantics:
    set:      set/update a single key. Preserves all other keys + formatting. prev_value returned.
    remove:   delete a single key. prev_value returned. No-op if key absent.
    merge:    patch multiple keys. null value deletes. Returns applied + removed maps.
    replace:  swap entire frontmatter block atomically. Destructive. HITL required.
              Returns prev_frontmatter so caller can roll back.

list_properties                      Aggregate frontmatter property usage across vault (which keys, what types, counts, sample values)
  acl:       read on root
  hitl:      never
  idem:      pure
  ratelimit: read
  input:     { vault, root?: VaultPath, sample_size?: number (default 5), ...Pagination }
  output:    PaginatedResponse({ key, types: string[], note_count, sample_values: unknown[] })
  errors:    acl_denied

find_notes_by_property               Filter notes by frontmatter values (typed comparators)
  acl:       read on root
  hitl:      never
  idem:      pure
  ratelimit: read
  input:     { vault, root?, filter: { key: string, op: "eq"|"neq"|"gt"|"gte"|"lt"|"lte"|"in"|"nin"|"contains"|"exists"|"absent", value?: unknown }[], ...Pagination }

  errors:    invalid_input, acl_denied
```

```
rewrite_link                         Rename a wikilink target across the vault (used by move_note internally; exposed for refactors)
  acl:       write on every modified note
  hitl:      required
  idem:      keyed
  ratelimit: bulk
  input:     { vault, from_target: string, to_target: string, scope?: "vault"|"folder", folder_path?: VaultPath, dry_run?: boolean (default true), elicit_token?: ElicitToken, idempotency_key? }
  output:    { from_target, to_target, files_modified: Array<{ path, occurrences }>, total_replacements, dry_run: boolean }
  errors:    acl_denied, elicit_required, read_only_mode

prune_hub_links                      Hub-note edge reduction. Reads a note, applies a caller-supplied policy to its outgoing links, drops links failing the policy
  acl:       write on path
  hitl:      required
  idem:      keyed
  ratelimit: write
  input:     { vault, path,
               policy: {
                 kind: "drop_below_score"
                     | "keep_top_n"
                     | "drop_unannotated_list_items"
                     | "drop_no_prose_context"
                     | "jsonlogic",
                 score_field?: string,            // kind=drop_below_score|keep_top_n
                 threshold?: number,              // kind=drop_below_score
                 n?: number,                      // kind=keep_top_n
                 min_annotation_chars?: number,   // kind=drop_unannotated_list_items
                 min_prose_chars?: number,        // kind=drop_no_prose_context
                 logic?: object                   // kind=jsonlogic; evaluated per link over link metadata
               },
               max_removals?: number (default 50),
               dry_run?: boolean (default true),
               elicit_token?: ElicitToken,
               idempotency_key }
  output:    { path, policy_used, links_before, links_after,
               removed: Array<{ target, reason, line }>,
               content_hash?: string, dry_run: boolean }
  errors:    note_not_found, invalid_input (bad policy shape), acl_denied, elicit_required

  Policy primitive semantics:
    drop_below_score:           drop any link whose score_field (read from link metadata
                                or per-link scoring callback) is below threshold.
    keep_top_n:                 keep the top n links by score_field, drop the rest.
    drop_unannotated_list_items: drop links that appear as bare list items with fewer than
                                min_annotation_chars of surrounding description text on the line.
    drop_no_prose_context:      drop links whose surrounding paragraph (excluding the link
                                token itself) is shorter than min_prose_chars.
    jsonlogic:                  escape hatch. JSONLogic expression evaluated against the link's
                                full metadata object: { target, line, col, in_codeblock,
                                in_list_item, list_item_text, prose_context, target_path,
                                target_backlink_count, target_outgoing_count }.

  Composition note: callers needing a multi-pass strategy (e.g., the vault-writer skill's
  prose/index/mixed classifier) invoke prune_hub_links multiple times, once per policy primitive,
  using dry_run on intermediate passes to compose a removal set before applying.
```

## Domain 6: Search (7 tools)

```
search_vault                         Unified search dispatch (mode router)
  acl:       read on root
  hitl:      never
  idem:      pure
  ratelimit: read
  input:     { vault, query: string|object, mode?: "auto"|"text"|"regex"|"dql"|"jsonlogic"|"semantic" (default "auto"), root?, explain?: boolean (default false), ...Pagination }
  output:    PaginatedResponse({ path, score: number, mode_used: string, chunk_id?: string, snippet?: string, line?: number }) & { _explain?: { modes_tried: string[], chosen: string, reason: string } }
  errors:    acl_denied, dql_error, jsonlogic_error, embedding_provider_error
  obs:       tags mode_used, modes_tried

  Note: when vault config sets `auto_mode_explain: "always"`, the _explain object is returned
```

## Cross-cutting items for G2.2-G2.5

These surfaced while drafting G2.1 and need resolution in later G2 sub-docs. They do not invalidate G2.1; they tighten things G2.1 has only sketched.

1. **Plugin-bridge ACL needs its own scope class.** `execute:<plugin>` as a first-class scope in the JWT model. The G2.1 spec uses `execute on <bridge>` informally in several tools (dataview, templater, quickadd, ocr, excalidraw, workspaces, bookmarks, memory bridge, capture bridge, command bridge). Decide in G2.4 Security: either add `execute:<plugin>` as a real scope OR collapse to "if you can read, you can execute read-only plugin ops" for read-side and "if you can write, you can execute write-side plugin ops" for write-side. Cleaner path is the explicit scope.

2. **Token-budget governor across response shapes.** `bundle_folder` caps at 500KB; `list_notes` paginates; `search_*` paginates; `read_notes` caps at 50 paths. Inconsistent governors. For G2.4 we should pick a single response-byte ceiling that applies after pagination clamping, and surface `overflow_bytes: true` when output got truncated for size rather than count. Right now a single huge note via `read_note` has no governor at all, which is a footgun for autonomous agents.

3. **Idempotency key TTL.** Spec uses "24h" for the cache; G2.3 Storage needs to lock the exact value and whether it's per-tool tunable.

4. **HITL elicit token lifetime.** `elicit_required` errors return an `elicit_token`. How long valid? Single-use vs reusable within window? Decide in G2.4.

5. **Materialization of memory entities.** Domain 22 entities can optionally materialize as `.md` files. Default is `materialize: true` but the memory folder path needs to be per-vault config. G2.3 Storage and G2.4 Security both touch this.

6. **`safe_commands` allowlist for `execute_command`.** Need to seed a reasonable default list. G2.4 Security.

7. **Companion plugin discovery vs explicit declaration finalization.** Q3 from G2.1 conventions settled on hybrid auto-probe; G2.2 Architecture needs to spec the probe endpoint contract and config-override schema precisely.

8. **HITL threshold policy location.** Several tools encode "required if X > N" rules (`delete_attachment` if `reference_count > 0`, `update_canvas` if removing >10 nodes, `bulk_create_notes` if count >50, `update_task` if status flip on >7d-old done, `move_attachment` if `reference_count > 10`). Where do the threshold values live? Two options for G2.2 Architecture to pick between:
    - **Hardcoded in tool implementations.** Brittle but simple. Threshold values live in code.
    - **Per-vault config under `hitl_thresholds:` key.** Operators can tune for agent vs human use. Matches the existing `elicit:` config-block pattern in G2-architecture.
   The decision shapes where in the dispatch path policy evaluation happens (tool function vs auth/policy layer).

## V1 → V2 forward-compat hooks already baked in

- Every search/read tool returns `chunk_id` where applicable (V2 substrate)
- `search_semantic` returns `score` (V2 RRF input)
- `read_note` and `read_notes` return `content_hash` (V2 incremental indexing)
- `workspace_memory` JSONL traces include `tool`, `caller`, `args_hash` (V2 retrieval training data)
- `memory_entities` schema separable from chunk schema (V2 fuses them at retrieval time)
- `write_note` `mode_used` and `prev_hash` on output (V2 change-tracking + audit log)
- `update_frontmatter` `prev_value` / `prev_frontmatter` (V2 audit log + undo support)

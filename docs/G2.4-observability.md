# Observability Model (G2.4)

obsidian-tc emits three observability streams: OTEL traces, Prometheus metrics, and MORGIANA CloudEvents. Each is opt-in beyond a sensible local default, and none of them block tool execution if the export path fails.

## Pillars

| Stream            | Purpose                                                       | Default                            |
| ----------------- | ------------------------------------------------------------- | ---------------------------------- |
| OTEL traces       | Per-call structured spans for debugging and latency analysis. | stdout in dev; off in production.  |
| Prometheus metrics | Aggregate counters, histograms, gauges for SLOs and alerting. | Endpoint disabled (`:0`).          |
| MORGIANA events   | Structured CloudEvents for security and ops audit.            | Local JSONL spool, on by default.  |
| JSONL traces      | Full per-call replay corpus (specified in G2.3 D8).           | On by default per vault.           |
| `event_log` table | Local SQLite audit table (specified in G2.3 D9).              | On by default per vault.           |

## OTEL traces

### Span model

One root span per tool call. Span name: `obsidian_tc.<tool_name>`. Span kind: `SERVER`.

Verbose mode (`vault.observability.trace_detail: "verbose"`) adds child spans:

| Child span                     | Captures                                          |
| ------------------------------ | ------------------------------------------------- |
| `obsidian_tc.auth_check`       | JWT validation                                    |
| `obsidian_tc.acl_eval`         | Scope check, folder ACL evaluation                |
| `obsidian_tc.policy_eval`      | HITL, idempotency, rate limit                     |
| `obsidian_tc.tool_impl`        | Actual tool execution                             |
| `obsidian_tc.output_serialize` | Response serialization, governor truncation       |

Default mode emits the root span only.

### Attributes

```
service.name                  = "obsidian-tc"
service.version               = <semver>
service.instance.id           = <pid + start_ts>
obsidian_tc.vault_id          = <vault>
obsidian_tc.tool              = <tool_name>
obsidian_tc.caller_hash       = <sha256(caller_id)[:8]>
obsidian_tc.scopes_required   = <comma-list of scope strings>
obsidian_tc.status            = "ok" | "denied" | "error"
obsidian_tc.error_code        = <code | absent>
obsidian_tc.elicit_used       = bool
obsidian_tc.idem_hit          = bool
obsidian_tc.overflow_b        = <number, 0 if no truncation>
obsidian_tc.duration_ms       = <number>
rate_limit.hit                = bool
```

Caller identity is hashed to 8 hex chars to bound cardinality.

### Sampling

`vault.observability.traces_sample_rate: 1.0` is the V1 default (laptop scale, traces are cheap). Configurable per vault. Error spans (status != "ok") are always sampled regardless of rate.

### Export

OTLP HTTP. Configured per vault:

```yaml
observability:
  traces_sample_rate: 1.0
  trace_detail: standard           # or "verbose"
  otel:
    endpoint: "http://localhost:4318"   # OTLP HTTP receiver
    headers:
      Authorization: "Bearer ${OTEL_TOKEN}"
```

Default in dev: emit to stdout (no remote endpoint). Default in production: off unless `observability.otel.endpoint` is set. MORGIANA can be the configured endpoint.

## Prometheus metrics

### Catalog

**Counters:**

| Name                                            | Labels                                |
| ----------------------------------------------- | ------------------------------------- |
| `obsidian_tc_tool_calls_total`                  | `vault`, `tool`, `status`             |
| `obsidian_tc_acl_denied_total`                  | `vault`, `scope_class`, `reason`      |
| `obsidian_tc_hitl_elicited_total`               | `vault`, `tool`                       |
| `obsidian_tc_idempotency_hits_total`            | `vault`, `tool`                       |
| `obsidian_tc_idempotency_cache_skipped_total`   | `vault`, `tool`                       |
| `obsidian_tc_rate_limit_hits_total`             | `vault`, `scope_class`                |
| `obsidian_tc_governor_truncations_total`        | `vault`, `tool`                       |
| `obsidian_tc_morgiana_emit_dropped_total`       | `vault`, `reason`                     |

**Histograms:**

| Name                                  | Labels            | Buckets                                       |
| ------------------------------------- | ----------------- | --------------------------------------------- |
| `obsidian_tc_tool_duration_seconds`   | `vault`, `tool`   | Default (Prometheus client library defaults). |
| `obsidian_tc_response_bytes`          | `vault`, `tool`   | 1k, 10k, 100k, 1M, 10M (log-spaced).          |

**Gauges:**

| Name                                  | Labels   |
| ------------------------------------- | -------- |
| `obsidian_tc_active_sessions`         | `vault`  |
| `obsidian_tc_capture_queue_depth`     | `vault`  |
| `obsidian_tc_elicit_tokens_pending`   | `vault`  |
| `obsidian_tc_idempotency_cache_bytes` | `vault`  |

### Endpoint

Server process exposes `/metrics` on `:metrics_port`. Default `:0` (disabled). Bind localhost only by default; non-localhost bind triggers the same hardcoded JWT requirement as MCP transport (G2.2 commitment 8).

```yaml
observability:
  prometheus:
    enabled: true
    port: 9464
    bind: "127.0.0.1"
```

### Cardinality notes

Caller hash is deliberately omitted from Prometheus labels. Prometheus' cardinality budget is tighter than OTEL's, and caller-dimension breakdowns belong in traces and MORGIANA events. Use scope class instead of full scope strings for the same reason.

## MORGIANA events

### Envelope

CloudEvents 1.0:

```jsonc
{
  "specversion": "1.0",
  "id": "<uuid>",
  "source": "obsidian-tc/<vault_id>",
  "type": "<event_type>",
  "datacontenttype": "application/json",
  "time": "<rfc3339>",
  "data": { ... }
}
```

### Event types

| Type                          | Emission point                                          |
| ----------------------------- | ------------------------------------------------------- |
| `tc.tool.call.completed`      | Every tool call (success or failure).                   |
| `tc.acl.denied`               | Scope or path denial.                                   |
| `tc.elicit.requested`         | Policy layer issued an elicit token.                    |
| `tc.elicit.consumed`          | Caller consumed an elicit token.                        |
| `tc.rate_limit.hit`           | Rate limit refused a call.                              |
| `tc.governor.overflow`        | Response-byte governor truncated or refused output.     |
| `tc.vault.cache_reset`        | `reset_vault_cache` was invoked.                        |
| `tc.server.start`             | Server process started, vaults registered.              |
| `tc.server.shutdown`          | Server process exiting (SIGTERM or natural stop).       |

### Data shape

```jsonc
{
  "vault_id":        "<vault>",
  "tool":            "<name | null>",
  "caller_hash":     "<8char>",
  "scopes_required": ["..."],
  "status":          "ok | denied | error",
  "duration_ms":     <number | null>,
  "elicit_token":    "<jti | null>",
  "idempotency_key": "<key | null>",
  "result_size":     <bytes | null>,
  "overflow_bytes":  <bytes | null>,
  "error":           { "code": "...", "message": "..." } | null,
  "count":           <number | null>     // bulk batches; otherwise null
}
```

Bulk operations emit one event per batch with `count` set. No per-row emission inside hot loops.

### Transport

V1 default: local JSONL spool. MORGIANA tails the file.

```
<cache_dir>/<vault>/morgiana-events.jsonl
```

Daily rotation by date (matches G2.3 D8 trace file layout). Retention `retention.morgiana_events_days: 90` (default, per vault). HTTP push is opt-in:

```yaml
observability:
  morgiana:
    spool: true                                            # default
    http_endpoint: "https://morgiana.internal/events"      # opt-in
    http_headers:
      Authorization: "Bearer ${MORGIANA_TOKEN}"
```

Spool and HTTP push can both be enabled. Either failing fails-soft.

### Failure mode

Spool write failure → log to `event_log`, increment `obsidian_tc_morgiana_emit_dropped_total{reason="spool_write_failed"}`, drop the event. HTTP push failure → log, increment counter with `reason="http_push_failed"`, fall back to spool if both are configured. **Tool execution never blocks on event emission.**

Crash-loss budget: 5 seconds (matches G2.3 D8 JSONL trace flush cadence).

### Schema sharing

The CloudEvents data schema lives in `packages/shared/` as a Zod schema. MORGIANA imports it and validates incoming events. Schema version is the `specversion` field; breaking changes bump it.

## Related local stores

Two pre-existing local stores from earlier gates support observability without being part of the export pipeline:

- **JSONL traces** (G2.3 D8). Full per-call replay corpus with optional field-level redaction. Default retention 90 days per vault. Source of the V2 retrieval-training corpus.
- **`event_log` table** (G2.3 D9). SQLite audit table for server-internal events including sweep results and emit-drop counts. Default retention 30 days per vault. Source of truth for any forensic question that crosses the OTEL/Prometheus/MORGIANA boundary.

## Configuration summary

```yaml
observability:
  trace_detail: standard                # or "verbose"
  traces_sample_rate: 1.0
  otel:
    endpoint: null                      # set to enable OTLP HTTP
    headers: {}
  prometheus:
    enabled: false
    port: 9464
    bind: "127.0.0.1"
  morgiana:
    spool: true
    http_endpoint: null
    http_headers: {}
  retention:
    morgiana_events_days: 90
    traces_days: 90                     # G2.3 D9
    event_log_days: 30                  # G2.3 D9
```

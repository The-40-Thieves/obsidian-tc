# Security Model (G2.4)

obsidian-tc treats its security surface as a first-class deliverable. This document specifies the authentication model, authorization model, runtime governance, and refusal taxonomy. Every dangerous capability is either gated by explicit operator opt-in or fenced by a hardcoded floor that vault configuration cannot override.

## Threat model

**Targets to protect:** vault content (notes, attachments), agent-managed memory (memory_entities, capture queue), operator credentials, and the host system where obsidian-tc runs.

**Attackers in scope:**

- Compromised caller (stolen JWT, malicious agent with valid token).
- Malicious vault content (a note that tries to elicit, capture data via plugin bridges, or escape via templates).
- Plugin-level RCE (Templater run, QuickAdd, Smart Connections proxying caller input into arbitrary code).
- Network attacker on a non-localhost deployment.

**Out of scope (V1):**

- Adversarial Obsidian itself (we trust the Obsidian process and its Local REST API plugin).
- Adversarial filesystem (we trust OS path resolution).
- Adversarial npm/cargo registry (we trust transitively reviewed dependencies).

## Authentication

### Transport posture

Two transport modes. STDIO for local single-user use. Streamable HTTP for remote agents.

Auth modes:

- `none`: no auth. **Hardcoded refusal** to bind non-localhost addresses with this mode (G2.2 commitment 8). Source-code-level refusal, not config-overridable.
- `jwt`: required for any non-localhost deployment. The default for HTTP transport.
- `oidc`: reserved for V1.x, not implemented.

### Token issuance

JWTs are minted by the operator via CLI:

```bash
obsidian-tc auth issue \
  --vault personal-vault \
  --scopes 'read:* write:notes write:meta bulk:*' \
  --ttl 7d \
  --caller "agent-claude-2026-05-19"
```

The CLI emits the token to stdout. The operator delivers it to the caller out-of-band. Server self-issuing is not supported in V1; a server `/auth/issue` admin endpoint would make any compromised admin token a total-compromise pivot.

### Token shape

```jsonc
{
  "iss": "<vault_id>",
  "sub": "<caller_id>",
  "aud": "obsidian-tc",
  "vault_id": "<vault_id>",
  "scopes": ["read:notes", "write:notes", "bulk:*"],
  "iat": 1758268800,
  "exp": 1758873600,
  "jti": "<uuid>"
}
```

Signing: HMAC-SHA-256 (HS256). Signing key at `<config_dir>/signing.key`, mode 0600 enforced at startup. Generated by `obsidian-tc auth init` on first run.

### Key rotation

```bash
obsidian-tc auth rotate-key
```

Generates a new signing key, archives the previous one with a `.rotated.<timestamp>` suffix, and immediately invalidates all existing tokens. The operator reissues new tokens to callers. No grace period in V1; V1.x may add a dual-key rotation window.

### Forward-compat slot

External IdP via OIDC (Cloudflare Access, Auth0, etc.) is reserved as a V1.x slot. The `auth.mode` config field accepts `jwt`, `none`, or (reserved, not yet implemented) `oidc`.

## Authorization

Authorization runs in ACL layer L3 of the dispatch pipeline (located by G2.2 commitment 2). Three checks in order: scope check, bulk-class check, path check.

### Scope catalog

Verb-family × domain-bucket. Flat snake_case strings. About 30 scopes total.

| Family        | Scope               | Covers                                                                                |
| ------------- | ------------------- | ------------------------------------------------------------------------------------- |
| Read          | `read:notes`        | `read_note`, `read_multiple_notes`, list notes, link-graph reads (Domain 5)           |
|               | `read:meta`         | frontmatter, tags, links, backlinks, orphans                                          |
|               | `read:search`       | `search_vault` and mode variants                                                      |
|               | `read:bases`        | `bases_query`, `bases_list`                                                           |
|               | `read:canvas`       | canvas read ops                                                                       |
|               | `read:dataview`     | Dataview query ops                                                                    |
|               | `read:tasks`        | Tasks plugin reads                                                                    |
|               | `read:memory`       | memory_entities reads                                                                 |
|               | `read:capture`      | capture queue reads                                                                   |
|               | `read:traces`       | JSONL trace reads                                                                     |
|               | `read:vault`        | vault registry, server status                                                         |
| Write         | `write:notes`       | create/update/delete notes, append, prepend, patch                                    |
|               | `write:meta`        | frontmatter/tags/link mutations                                                       |
|               | `write:bases`       | base file mutations                                                                   |
|               | `write:canvas`      | canvas mutations                                                                      |
|               | `write:tasks`       | Tasks plugin mutations                                                                |
|               | `write:memory`      | memory entity create/update/delete incl. materialization                              |
|               | `write:capture`     | capture queue add/commit                                                              |
|               | `write:periodic`    | daily/weekly note creation                                                            |
|               | `write:templater`   | Templater template execution (plugin can run arbitrary code)                          |
| Admin         | `admin:vault`       | `add_vault`, `remove_vault`, `reset_vault_cache`, `reload_vault`                      |
|               | `admin:server`      | lifecycle, config reload                                                              |
|               | `admin:plugins`     | probe override, force-enable/disable                                                  |
|               | `admin:auth`        | `obsidian-tc auth *` operations issued via tool surface                               |
| Parameterized | `execute:<plugin>`  | plugin-bridge ops; one scope per discovered plugin                                    |
|               | `execute:shell`     | `execute_command` (separate from plugin family)                                       |
| Bulk          | `bulk:*`            | required on top of underlying scope for any `bulk_*` tool                             |

Hierarchy: family wildcard (`read:*`, `write:*`, `admin:*`, `execute:*`) satisfies any scope in that family. Sibling scopes do not imply each other.

### Folder ACL

Glob patterns, last-match-wins. Default ruleset:

```yaml
folder_acl:
  defaults:
    - allow: "**"
    - deny: ".obsidian/**"
    - deny: ".trash/**"
    - deny: ".git/**"
    - deny: "**/_archive*/**"
  per_scope:
    write:templater:
      - deny: "**"           # opt-in only; operator must add explicit allows
    write:notes:
      - inherits: defaults
    read:notes:
      - inherits: defaults
```

Path resolution: the folder ACL gates the **symlink-canonical** (realpath-resolved) vault-relative path per request, plus an **inode guard** — a hard-linked regular file (`st_nlink > 1`) is rejected under an ACL (a hard link aliases an inode realpath cannot dereference), reads run on the opened fd, and the atomic write opens its temp file `O_EXCL | O_NOFOLLOW` on a random name (v1.3.2), and the .obsidian/.git/.trash default-deny folds case so a case-variant control-directory path cannot evade it on a case-insensitive filesystem (v1.3.3). config paths use `realpath` at startup per G2.2. Refusal: `path_denied` with `details: { path, scope, rule_id }`.

### Scope evaluation order

1. Validate JWT signature and expiry (Transport L2).
2. Resolve required scopes for tool from G2.1 spec.
3. Check each required scope against token scopes with hierarchy expansion.
4. Check `bulk:*` if tool is a bulk variant.
5. Resolve all paths in args via canonical path rules.
6. Run folder ACL per path against scope-specific rules.
7. On pass: dispatch to Policy layer L4c for HITL evaluation.

Refusal taxonomy:

- `scope_denied`: token lacks required scope.
- `bulk_scope_required`: tool is bulk variant; token has underlying scope but lacks `bulk:*`.
- `path_denied`: folder ACL rejected one or more paths.

## HITL governance

The elicit primitive (G2.2 commitment 5) is the human-in-the-loop checkpoint for actions that warrant operator confirmation. Tokens are minted by Policy layer L4c, returned via MCP elicitation, and consumed on the caller's confirmation call.

### Elicit token lifecycle

| Attribute | Value                                                          |
| --------- | -------------------------------------------------------------- |
| Format    | 16 bytes random, hex-encoded (32 chars). `crypto.randomBytes`. |
| TTL       | 5 minutes (G2.2 bonus).                                        |
| Use       | Single-use. First-consume-wins-redemption.                     |
| Binding   | Bound to `(vault, tool_name, args_hash, caller)` (G2.3 D1; caller binding v1.3.2).                   |
| Storage   | `elicit_tokens` table per G2.3 D1.                             |
| Sweep     | 60s fast-sweep via `setInterval` (G2.3 D9).                    |

### Thresholds and floors

Tool-level HITL class declared in G2.1 spec is the floor. Vault config may raise (more cautious) but not lower below the floor for high-risk scopes.

Hardcoded floors (cannot be lowered):

| Pattern                  | Floor             |
| ------------------------ | ----------------- |
| `execute:*` family       | `always_elicit`   |
| `bulk_delete_*` tools    | `always_elicit`   |
| `admin:auth` scope       | `always_elicit`   |
| `write:templater` scope  | `always_elicit`   |

Vault config:

```yaml
hitl_thresholds:
  bulk_delete_notes: { mode: always_elicit }       # cannot lower
  bulk_create_notes: { mode: threshold, gt: 50 }   # raised from spec default
  read_note: { mode: never }                       # ok to lower; read is non-destructive
```

Refusal-to-lower at server startup: `hitl_floor_violation` error naming the scope and the floor value.

## Runtime governance

### Response-byte governor

Single byte cap shared with idempotency cache (G2.3 D2). Default `1048576` (1 MB). Per-tool override supported.

Behavior by tool class:

| Class                                  | Behavior                                                                                           |
| -------------------------------------- | -------------------------------------------------------------------------------------------------- |
| Pagination-aware (search, list, query) | Refuse with `response_too_large`; `details` includes `suggested_limit` and `current_bytes_estimate`. |
| Read-single (`read_note`, `get_attachment`) | Truncate output; response includes `truncated: true` and `overflow_bytes: <N>`.               |
| Bulk variants                          | Refuse if total > cap; `details` suggests smaller batch.                                           |

Idempotency replays return the same truncated result with the same `overflow_bytes` flag — replay is deterministic across the governor. The `result_size` column on `idempotency_keys` (G2.3 D2) is the pre-truncation size; the governor reads it for replay sizing.

Config:

```yaml
response_governor:
  max_bytes: 1048576              # default; matches idempotency.max_result_bytes
  per_tool:
    read_note: 5242880            # 5 MB override
    get_attachment: 10485760      # 10 MB override
```

### Idempotency error taxonomy

Errors (call rejected):

| Code                        | Cause                                                                         |
| --------------------------- | ----------------------------------------------------------------------------- |
| `idempotency_key_mismatch`  | Same key, different `args_hash`. Hint: stored `args_hash[:8]`.                |
| `idempotency_in_flight`     | Same key currently executing in another caller. Caller should retry.          |

States (logged, behavior = forward to fresh execution):

| Code                          | Cause                                                                          |
| ----------------------------- | ------------------------------------------------------------------------------ |
| `idempotency_cache_skipped`   | Original result exceeded `max_result_bytes`; cache was never written.          |
| `idempotency_key_expired`     | Key TTL elapsed (default 24h, G2.3 D2/D5).                                     |

In-flight detection: `idempotency_keys.started_at` and `completed_at` columns. Row with `started_at IS NOT NULL AND completed_at IS NULL` is in-flight. `result` and `result_size` are nullable (NULL while in-flight, populated on completion). Lock TTL 60s; rows reaped by the existing 60s fast-sweep (G2.3 D9).

V1 stance: "replay is best-effort. Callers should not rely on referenced state (note IDs, attachment paths, etc.) being live at replay time." Replay-drift handling deferred to V1.x.

### `execute_command` governance

Default mode `production`: tool returns `execute_command_disabled` regardless of token scopes.

Development mode:

```yaml
execute_command:
  mode: development
  allowlist:
    - "git status"
    - "git log --oneline -10"
    - "ls"
  hitl: always_elicit          # floor; cannot lower
```

Each allowlist entry is exact-match on command + args. No globs.

Hardcoded denylist applied first regardless of allowlist (shell-metacharacter denial): `rm`, `mkfs`, `dd`, `>`, `|`, `;`, `&&`, `||`, `$(`, backtick. Presence of any denylist token in the input string → refuse.

Refusal taxonomy:

| Code                         | Cause                                                                      |
| ---------------------------- | -------------------------------------------------------------------------- |
| `execute_command_disabled`   | Mode is `production`.                                                      |
| `command_not_allowlisted`    | Mode is `development` but command is not in the operator allowlist.        |
| `command_unsafe`             | Command contains a hardcoded-denylist token.                               |

### Rate limits

Token bucket per `(caller_hash, scope_class, vault_id)`. In-memory only (Bun process); restart resets buckets.

Tiered defaults:

| Scope class | Rate       | Burst |
| ----------- | ---------- | ----- |
| `read:*`    | 600 / min  | 100   |
| `write:*`   | 60 / min   | 20    |
| `bulk:*`    | 10 / min   | 3     |
| `execute:*` | 5 / min    | 1     |
| `admin:*`   | 5 / min    | 1     |

Backoff signaling: MCP error response includes `retry_after_seconds`. OTEL span attribute `rate_limit.hit: true`. Prometheus counter `obsidian_tc_rate_limit_hits_total{vault, scope_class}` increments.

Refusal: `rate_limit_exceeded` with `details: { scope_class, retry_after_seconds, current_burst, current_rate }`.

Distributed deployments are out of scope for V1 (G2.2 commitment 10 = colocated topology). Multi-process rate limiting deferred to V1.x.

## Refusal taxonomy summary

Single table for caller-facing error codes from the security and governance layers:

| Code                          | Layer        | Meaning                                                  |
| ----------------------------- | ------------ | -------------------------------------------------------- |
| `auth_token_invalid`          | Transport L2 | JWT signature, expiry, or audience mismatch.             |
| `auth_token_revoked`          | Transport L2 | JWT `jti` is in the revocation list.                     |
| `scope_denied`                | ACL L3       | Token lacks required scope.                              |
| `bulk_scope_required`         | ACL L3       | Bulk variant tool; token lacks `bulk:*`.                 |
| `path_denied`                 | ACL L3       | Folder ACL rejected one or more paths.                   |
| `hitl_required`               | Policy L4c   | Elicit token required; caller must consent.              |
| `hitl_token_invalid`          | Policy L4c   | Elicit token expired, mismatched, or already consumed.   |
| `idempotency_key_mismatch`    | Policy L4c   | Same idempotency key, different args.                    |
| `idempotency_in_flight`       | Policy L4c   | Same idempotency key currently executing.                |
| `rate_limit_exceeded`         | Policy L4c   | Bucket exhausted.                                        |
| `response_too_large`          | Output L5    | Pagination-aware tool refused; caller should `limit`.    |
| `execute_command_disabled`    | Tool impl    | Production mode; `execute_command` is off.               |
| `command_not_allowlisted`     | Tool impl    | Dev mode; command not in operator allowlist.             |
| `command_unsafe`              | Tool impl    | Hardcoded-denylist token in input.                       |
| `hitl_floor_violation`        | Startup      | Config lowered HITL below hardcoded floor.               |

## Operational notes

- Signing key file mode 0600 enforced at startup. Server refuses to start if mode is more permissive.
- All security-relevant events emit MORGIANA events (`tc.acl.denied`, `tc.elicit.*`, `tc.rate_limit.hit`). See `docs/G2.4-observability.md`.
- No telemetry leaves the host unless explicitly configured (OTEL endpoint, Prometheus scrape from a remote, MORGIANA HTTP push).
- `obsidian-tc auth list` (CLI) shows active token JTIs and their scopes; useful for forensics after a suspected compromise.
- `obsidian-tc auth revoke <jti>` (V1.x) deferred. V1 revocation is global only (`rotate-key`).

## CLI surface

```
obsidian-tc auth init                 # generate signing key (first run)
obsidian-tc auth issue <flags>        # mint a JWT
obsidian-tc auth rotate-key           # invalidate all tokens, generate new key
obsidian-tc auth list                 # show active JTIs (issued, not expired)
obsidian-tc auth revoke <jti>         # V1.x; deferred
```

## G3 amendments (2026-05-26)

Applied after G3 Simplicity Check gate (THE-141). Changes locked:

- `read:graph` scope removed from catalog. Link-graph reads (Domain 5) are now covered by `read:notes`. Segregating a separate scope added ACL surface without meaningful security benefit — all link-graph operations read the same underlying note content that `read:notes` already governs.
- `read:notes` coverage updated: link-graph reads (Domain 5) explicitly noted. This covers `get_outgoing_links`, `get_backlinks`, `find_orphans`, `find_unresolved_links` (Domain 5).
